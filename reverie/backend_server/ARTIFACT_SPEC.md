# Artifact Content Generation — Phase 2 Spec

## Overview

Currently, `ArtifactManager.create_artifact()` stores a `content_summary` (max 300 chars) generated by the LLM. Phase 2 adds a `content_full` field with rich, multi-paragraph content appropriate to each artifact type, generated via `free_form=True` LLM calls.

## Existing System Summary

- **artifact_manager.py** — `ArtifactManager` class with `create_artifact()`, persistence to `{sim_folder}/artifacts/artifacts.json`
- **reverie.py:268-348** — `try_create_artifact()` calls `run_gpt_prompt_create_artifact()` for name/description/content_summary/quality, then calls `create_artifact()`
- **gpt_structure.py** — `_llama_cpp_generate(prompt, free_form=True)` for unconstrained text, `ChatGPT_request()` wrapper
- **run_gpt_prompt.py:3171-3231** — `run_gpt_prompt_create_artifact()` generates artifact properties

## Content Types

| Artifact Type | Generator Function | Output |
|---|---|---|
| book, letter, essay, poem, journal, writing | `generate_written_content()` | 200-400 word prose/poetry |
| painting, drawing, sketch, artwork | `generate_painting_prompt()` | Rich image-generation prompt |
| song, music, composition | `generate_song_lyrics()` | Song lyrics or musical description |
| meal, food, recipe, cooking, baking | `generate_meal_description()` | Ingredients, preparation, occasion |
| invention, craft, build | `generate_invention_description()` | Technical/creative description |

## Keyword Matching

Action description is lowercased and checked for substrings (first match wins):

1. writing/book/letter/essay/poem/journal → written_content
2. paint/draw/sketch/artwork → painting_prompt
3. song/music/compos/sing → song_lyrics
4. cook/bake/meal/food/recipe → meal_description
5. build/invent/craft → invention_description

## LLM Prompt Templates

Each prompt includes: agent name, persona description, action description, artifact name/type. Uses `_llama_cpp_generate(prompt, free_form=True)` via a helper that mirrors `ChatGPT_request` but forces `free_form=True`.

### Written Content Prompt
```
{agent_name} is {action}. {persona_description}.
Write the full text of what {agent_name} wrote. 200-400 words.
Write in {agent_name}'s voice and personality. No meta-commentary.
```

### Painting Prompt
```
{agent_name} is {action}. {persona_description}.
Describe this artwork as a detailed image generation prompt: subject, style, colors, mood, composition. 2-3 sentences.
```

### Song Lyrics Prompt
```
{agent_name} is {action}. {persona_description}.
Write the complete song lyrics that {agent_name} composed. Include verses and chorus. 100-300 words.
```

### Meal Description Prompt
```
{agent_name} is {action}. {persona_description}.
Describe the meal: dish name, key ingredients, preparation method, aroma, presentation, and the occasion. 100-200 words.
```

### Invention Description Prompt
```
{agent_name} is {action}. {persona_description}.
Describe what {agent_name} built or invented: what it does, how it works, materials used, and why they made it. 100-200 words.
```

## Persistence

- Each generated content saved to `artifacts/{type}/{agent}_{timestamp}.txt`
- `content_full` field added to artifact dict in `artifacts.json` (persisted by existing `ArtifactManager.save()`)
- Append entry to `artifacts/artifact_log.jsonl`

## Edge Cases

- LLM failure: return empty string, log warning, artifact still created with just `content_summary`
- No keyword match: skip generation, log warning
- Filename sanitization: replace spaces/special chars in agent name with underscores
- Existing artifact without `content_full`: backward compatible (field simply absent)
